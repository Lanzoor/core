CSS Injection

This site intentionally injects CSS stylesheets via <link> tags using JavaScript.
There is nothing to worry about, but if you're curious about why this is done or how to troubleshoot it, keep reading.

When is CSS injection used, and why?

CSS injection is used for common components that appear across multiple pages.
Examples include the navigation panel, the link bar, and more.

In order to apply styles, a <link> tag must be present in the HTML document, hence the name. Manually adding <link> tags for shared components on every single page would be tedious and error-prone. It would also make updates painful whenever component locations change, which is exactly why we use CSS injection.

Why not just use a bundler?

Great question! This site uses a custom Vite configuration that doesn't support importing CSS directly within React components, so CSS injection serves as a practical workaround. We're actively working on a better setup that supports native CSS imports and more.

The styles are not loading!

Open the developer console (F12) and check for CSS injection logs. You can also search for the relevant file directly in our source code. If the issue persists, feel free to get in touch with details about what you're seeing.
